# Contributing

Thanks for helping Duggan Community League projects stay useful, safe, and maintainable.

## Workflow

1. Create a branch for the change.
2. Keep changes small and focused.
3. Open a pull request into the protected default branch.
4. Include what changed, how it was tested, and any rollback notes.
5. Wait for review or explicit maintainer approval before merging.

## Safety Rules

- Do not commit secrets, private member data, exported lists, payment data, or production credentials.
- Do not trigger production writes, app-store submissions, push notifications, email/SMS/social sends, member verification/sync/export, or third-party account changes unless the organization owner explicitly approves the action.
- Use local or staging evidence for tests whenever possible.
